# GitHub Desktop Quick Guide: Push & Pull a Repository

## Clone a Repository

1. Open **GitHub Desktop**.
2. Select **File → Clone Repository**.
3. Choose the repository from GitHub or paste its URL.
4. Select a local folder and click **Clone**.

## Pull the Latest Changes

Before you start working:

1. Open the repository in **GitHub Desktop**.
2. Click **Fetch origin**.
3. If updates are available, click **Pull origin** to download the latest changes.

## Make Changes

1. Edit your files in your preferred code editor.
2. Return to **GitHub Desktop** to review your changes.

## Commit Your Changes

1. Enter a brief **Summary** describing your changes.
2. (Optional) Add a longer description.
3. Click **Commit to main** (or your current branch).

## Push Your Changes

1. After committing, click **Push origin**.
2. Your changes will be uploaded to GitHub.

## Recommended Workflow

1. **Pull** the latest changes.
2. Make your edits.
3. **Commit** your changes.
4. **Push** your commits to GitHub.

Following this order helps reduce merge conflicts and keeps your local repository up to date.
