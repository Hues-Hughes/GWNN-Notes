			Data Protection
				- Data at Rest - Encrpytion
				- Data in Transit - Encrypting/Cryptography & Hashing
				- Data in Process - Most difficult to protect. Host machine lockdown, isolated envs, RAM/resources security
				
				Encrpytion
				
					Symmetric and Asymmetric (Single Shared Key vs Public vs Private Key)
					
						Symmetric - fast, for bulk data and VPN
							Examples - DES (Original) 3DES(DES applied 3 times) AES(fast and common) RC4 (Stream/other ciphers)
								note: DES/3DES is older and now unreliable
						
						Asymmetric - slower, used for small amounts of data.
							
							Examples/Protocols - RSA, DSA, ElGamal, ECC. IKE,SSH,SSL/TLS,HTTPS,IPsec.
							
							Goals - Confidentiality, Authentication, Integrity
							DH Key Exchange - used for when keys need to be exchanged over proviate channels.
							
						Obscuring Data Techniques
						
							- Data Masking - Substition, Shuffling, Nulling
							
							- Stegonagraphy - Conceals message within another file by altering least significant bits of pixels
							  
						Integrity and Authenticity:
						
							Hash Algorithsm & HMAC
								Keyed-Hash Message Auth(technically symmetric) Sender: hash(message and key) digest sent with message. Receiver: verifies with same key to recompile data.
