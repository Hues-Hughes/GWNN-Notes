	ACL
	
	When Packets match an ACL router does permit or deny action and remaining ACEs are dropped
	IPv4
		Standard ACL
			100-199 (may need to correct)
			Filters at Layer 3 w/ source IPv4 address only
			**Close to Dest.**
		Extended ACL
			2000-2699 (may need to correct)
			Filters at Layer 3 by Source and Dest + Layer 4 detail
			**Close to Source**
		Named/Numbered ACL
			can filter by ports and protocols as well
			eq is equal
			Uses keywords like SURFING (inbound) & BROWSING (outbound)
		
		Access-list apply {acl-number}{interface} command to apply ACL to router interface
		
	IPV6
		All IPv6 ACLs are NAMED
		Contains an implicit deny by default
		NAs(Neighbor Advertisement) and NSs(Solicitations) must be explicitly permitted to support Neigbor Discovery Protocol
		ipv6 traffic-filter {acl-name}{in|out}
	
	Wildcard Mask
		32 Bit Value paired with IP Address
		All Occurring within ACE
		Inverse of Subnet Mask
		More focused on working in tandem with ACLs
		0s mean match, 1 means ignore
		how to get: subnet mask - 255.255.255.255 = wildcard mask
	
	TCP flag bits SYN and FIN must be set to match returning traffic
