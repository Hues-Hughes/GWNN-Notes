
ALERT ALERT

	Types
		
		Security Onion
			- Open Source suite of Network Security Monitoring (NIDS)
			- 3 functions: Full Packet Capture, Intrusion Detection, Alerting & Analysis
	
		  CapMe (Human Readable)
			  Pcap Transcripts: Web app that renders raw packet captures via tcpflow or Zeek
		  
		  Zeek (NIDS)
			  Behavioral Engine: uses policy scripts. Records detailed HTTP, DNS, and TCP network transactions
			  Can submit suspcious network attachments to malware sandboxes, shut down host for policy violation
			  Resource heavy and Steep Learning Curve
			  Passive Blocker, calls on different tools to execute.
		  
		  Wazuh (HIDS)
			  Endpoint Protection: Replaces OSSEC for endpoint-level
			Integrity & Audit: Integrity monitoring, vulner. detection and policy compliance
			Intergrates with Kibana
		  
		  Snort
			  Pulled Pork: Automated Rule manager. Updates rules based on new security practices from Snort
			  Additional Details:
				  Rule Header:
					  Action: Alert, Drop, Pass, Log
					  Protocol: IP, TCP, UDP, ICMP
					  Addressing & Direction
				  Rule Options:
					  MSG: text string displayed in the console alert
					  Content: Specific byte or string pattern to match inside payload
					  MetaData: References, classtype, unique SID (signature ID)
				Rule Sources & Rulesets
					GPL Ruleset - Older Rules created under GPLv2 license
					Emerging Threats - Open Source
					Cisco Talos - Fully certified and maintained by Talos threat intelligence. Subscription based
		
	Analysis Consoles
			
			Types:
		
				Sguil
					High level console. primary starting point for investigations
				
				ELSA
					Enterprise Log Search and Archive - delivers fast search across big log repos. Direct link to CapME
				
				Kibana
				
				Snort/Suricata
					Signature-based NIDS engine. Feeds alerts into Sguil and ELSA
					Suricata is active in IDS. able to drop packets
				
				
	How Alerts are Generated
	
		Triggered by NIDS, HIDS, assest discovery, trans logs and Syslog
		
		Metadata: Precise timestamps and reporting sensor IDs
		
		Priority Coding: Real-time color-coded events from Very Low to High Priority
		
		Five-Tuple ID:
			Source and Dest IP, Source and Dest Port, Transport Protocol
			
	Classifcations
		
		True Positive - alert and intrusion
		False Positive - alert and no intrusion (can be costly, signals alert rule refinement)
		True Negative - no alert and no intrusion
		False Negative - no alert and intrusion (most dangerous)
		
	Deterministic vs Probabilistic Analysis
	Known, Precise data vs Likelihood
	
	Probabilisitic is typically used in real-time
	Deterministic less so
