

**Monitoring Common Protocols**

	Syslog
		UDP
		Industry standard for collecting event data
		High value attack target, tamper log messages, flood servers to hide slow exfiltration
		
		Severity is calculated (Severity x 8) + Facility
		
		- **0 – Emergency (_emerg_)**: System unusable; e.g., kernel panic.
	    
		- **1 – Alert (_alert_)**: Immediate action required; e.g., database outage.
		    
		- **2 – Critical (_crit_)**: Severe conditions; e.g., hardware failure.
		    
		- **3 – Error (_err_)**: Application or service errors; e.g., failed authentication.
		    
		- **4 – Warning (_warning_)**: Potential issues; e.g., high CPU usage.
		    
		- **5 – Notice (_notice_)**: Significant but normal events; e.g., service restart.
		    
		- **6 – Informational (_info_)**: Routine messages; e.g., successful logins.
		    
		- **7 – Debug (_debug_)**: Detailed diagnostic data.
		
	NTP - Network Time Protocol (Clock Sync)
			Time Alignment
			Accurate timestamps help trace an exploit path
			Corrupting NTP destroys even correlation
	
	DNS Abuse
		Large Target due to permitted everywhere
		
		Methods:
		
			Encapsulation - hiding protocols inside queries
			Exfiltration - Malware encodes stolen data into long subdomains
		
		Detection Strategy:
			
			Abnormal subdomain lengths, randomly generated domains, queries with unusual volume. Most sysadmin use Cisco Umbrella to navigate it automatically
		
	
	HTTP & HTTPs
		
		Plaintext vulnerable to intercept
		
		Injections downloading malware (IFrame injections can happen through compromised web browsers)
		
		HTTPS can protect privacy but obscure packet details from security monitoring
		
		SSL Inspection is essential for visibility but adds heavy processing overhead
		
	
	Email Protocols (SMTP/POP3/IMAP)
	
		Threat Vector: Blended into normal flows to spread malware and execute exfiltration
		
		Monitoring: Retrospective tracking, attachment sandboxing, initial injection detection.
		
	
	ICMP
		Legit Usages
			Protocol for ping and traceroute
			Error messaging
			Path Discovery
			
		Malicious Exploit
			Maps active hosts, network topology
			Tunneling: Hiding stolen files inside payload fields
			DoS Attacks: Flooding target systems with packets
		
	
	ACLs
		
		Port Probing: Attackers scan to identify allowed ports and addresses
		Spoofing & Tunneling: Source IPs can be spoofed
		
	NAT/PAT Translation
		
		Attribution Challenge: Multiple users share one public IP, hiding internal host identity
		
		Breaks Network tracking due to IP address replacement
		
		Logging Requirement: Requires both translation logs to correlate external alerts with internal endpoints
		
		NAT & PAT can be corrupted
		
		Pre NAT & Pre PAT workflows
	
	
	VPN & Encryption
	
		Renders packet inspection unreadable to security controls. 
		Malware can be used to establish tunnels through encrypted channels, taking advantage of this
		
		Solutions to combat this are pricey
		
	P2P (Peer to Peer)
		When hosts act as both client and server, this can be exploited to circumvent firewalls, spread malware. Should be strictly prohibited.
	
	Tor Network
		Anonymized browsing
		Used by threat actors for C2 communications and obfuscating destination IPs
	
	Load Balancing
		Session Distribution: Spreads active sessions across servers
		
		Side Effect: Single User Transaction may originate from multiple IPs simultaneously.
	
		Analyst Impact: Legit traffic can trigger false positives like Impossible Travel (user jumping from one geolocation to another impossibly fast)

**Network Security Data**

	Types
		Alert Data
			IDS/IPS Alerts
				Genereated when traffic violates security rule or matches known exploit. (IDS just monitors and alerts, IPS monitors and blocks)
				Snort is a IPS
				
		Session & Transaction Data
			Session = Metadata
				5 key fields = TS UID Source/Destination IPs and ports
				Zeke IDS tool is used to protect
				
			Transaction = Exchange
				Session can span multiple transactions
				Web server access logs, packet transcripts. Richer and heavier than session data
		
		Full Packet Captures (FPC)
		
			Stores full packet headers and payloads (capable of capturing full http pages)
			Deployed on perimeters (DMZ) more than internal infra
			Gold Standard for Forensics
			Resource Heavy
		
		Statisical Data
			
			Derived from analzying traffic patterns to define normal operations.
			Helps establish a baseline to flag unexpected spikes in volume, off-hour transfers, or unusual destinations
			
	
	Host Logs and Window Events
		
		HIDS - Host Based Logs
			Monitors & Blocks Unauthorized modifications on Individual hosts
		
		Window Event Types
			
			Errors
			Warning
			Information
			Success/Failure Audits
		
	Server Logs & SIEM/SOAR
		
		Server Logs
			DNS most valuable because it's the first step
			
		SIEM - Security Information and Event Management (Splunk)
			Long term log storage and real time reporting
			
			Six Core Functions
				Log collection
				normalization
				correlation
				aggregation
				reporting
				compliance
		
		SOAR (Security Orchestration and Automated Response) - Orchestration and Automation platform. Extends SIEM with automated workflows and response scripts
	
	TCPDump & Cisco Device Logs
		
		Packet Analyzer
			Captures & analyzes network packets in real time
			Uses GUI tools like Wireshark
			Only collects essential info, saves on storage
			
		Cisco Infra Logging
			Generates structed event logs
			
	
	Proxy & DNS Logs (Cisco Umbrella)
	
		Occurs through web proxies: in between client and server. Logs all HTTP(S) URL requests
			Outbound data loss verification, analyze user behavior
			Prevent connections from known malicious domains
		
		
	
	Next-Gen Firewalls
		Layer 7 Inspection
		Combines Firewalls, IPS and threat intelligence in single engine
		Can be integrated with SOARs
