
	FireWalls
		Stateless vs Stateful
		
		Stateless
			Packet filtering, permits/denies traffice based on Layer 3 & 4 info
			Vulnerable to spoofing
		
		Stateful
			Versatile, Common
			Tracks complete state with state table
			Analzes traffic Layer 3, 4 & 5
			
		Proxy/Application Gateway Firewall 
			Layers 3, 4 ,5 ,7
	
	
	Common Security Architectures
	
		Private and Public
			SOHOs and small networks
		
		DMZ
			3rd segment of network inbetween public and private
				**investigate in tandem use with next gen firewalls sandboxing abilities**
			
		ZPFs
			CISCO IOS model
			Groups Router Interfaces sections of Virtual networks/ into security zones
			LANs by default aren't able to communicate between, zone pairs must be used.
			
			Should be designed on pen and paper **FULLY** before implementation
			Self Zone is represented by the router
			
			Design steps
				1 Determine Zones
				2 Establish Policies between zones
				3 Design phyiscal infrastructure
				4 Merge Traffic Requirements
			
			Transit Traffic:
				the travel between zones when S and D IP addresses don't match to Interface
				Each policy ends in 1 of 3 ways
				
				- Inspect
				- Pass (Most popular example = VPN Tunnel)
				- Drop
			
			
		Layered Defense
			Made up of the following potentially:
				- Network Core Security
				- Perimeter Security
				- Communication Security
				- Endpoint Security
